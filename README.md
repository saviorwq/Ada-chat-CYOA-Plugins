## 项目定位

`CYOA`（Choose Your Own Adventure）是一个以“**规则可编辑** + **叙事可约束** + **状态可模拟**”为核心的游戏插件。

它不是简单聊天皮肤，而是完整的玩法层：

- 剧情运行：章节、地点、场景、任务、状态推进
- 交互运行：行动/对话四选项、方向控制、可互动目标
- 规则运行：装备约束、材质影响、锁钥联动、契约联动
- AI 运行：严格叙事边界、RAG、模型微调、异常回复纠偏

---

## 核心能力

### 1) 可视化创作

- 支持编辑：属性、物品、装备、技能、任务、角色、场景、章节、地点、世界观等
- 支持地点连边 `locationEdges`、旅行回合、受限步态额外回合
- 支持装备叙事角色区分（如拘束具 vs 防护装甲）
- 支持附件机制（振动/电击/充气/真空等）与约束联动

### 2) 实时叙事游玩

- AI 流式回复（SSE）
- 固定四选项（2 行动 + 2 对话）
- 可互动面板（NPC / 设施）
- 方向控制器（八方向 + 上下 + 进入/离开）
- 地图快速到达（已知地点）

### 3) 严格框架叙事

- 叙事绑定：游戏名、故事简介、世界观、背景、章节、地点、在场角色、设施、技能、任务、属性
- 漂移检测：章节越界、地点越界、人物越界会触发纠偏重试
- 能力越权检测可放宽（记录但不强制纠偏），减少过度打断

### 4) 稳定与可维护

- `cyoa-game.js` 已收敛为兼容层，核心逻辑拆为独立模块
- 保存策略：后端失败自动回退 `localStorage`
- 右侧“AI 实时调试”可在游戏中直接调规则

## 快速开始

### 安装

1. 将插件目录 `plugins/cyoa/` 放入 Ada Chat 的 `plugins/`
2. 确认 `manifest.json` 可被宿主加载
3. 在插件管理启用 **CYOA 文字冒险游戏**

### 首次使用

1. 打开插件设置页，创建/导入一个游戏
2. 在编辑器补齐基础信息（章节、地点、角色、初始场景）
3. 保存后进入游戏
4. 通过右侧面板查看状态、地图、任务，并在 AI 实时调试面板增删规则

## 编辑器说明

### 核心数据建议最小闭环

- 至少 1 个章节
- 至少 1 个地点
- 至少 1 个场景（建议绑定章节+地点）
- 指定玩家角色
- 配置初始章节/初始场景（或可回退规则）

### 装备与限制建议

- 不要靠“槽位名”隐式推断效果，优先在装备数据里显式配置约束
- `no_hands` 与 `no_fingers` 区分要明确
- 口部/视觉/步态限制建议配合附件与姿态标签一起建模

### 地图旅行建议

- 使用 `locationEdges` 显式配置地点连通
- 为边配置 `travelTurns`
- 为受限步态配置 `limitedStepExtraTurns`（边级 > 游戏级 > 默认值）

## 运行时与 AI 机制

### 叙事输入帧（严格）

发送给 AI 的上下文会包含：

- 当前章节、地点、区域
- 在场角色清单及角色档案约束
- 玩家状态（姿态/技能/任务/属性/装备/限制）
- 世界设定文本（故事简介、世界观、背景）
- RAG 摘要 + 最近历史

### 选项生成

- 强制统一为 4 项
- 支持约束改写（例如手部受限时重写不合法动作）
- 前端按钮支持固定宽度与文本截断，保证布局稳定

### 漂移纠偏

- 章节偏航：纠偏
- 地点偏航：纠偏
- 人物越界：纠偏
- 能力越权：可记录不重试（根据当前策略）

## 右侧面板功能

当前右侧栏为顶层折叠结构，常见包含：

- 故事
- 属性
- 状态
- 物品
- 地图
- AI 实时调试
- 技能 / 任务 / 章节 / 存档

其中：

- **地图**：按区域分组、显示 ETA、仅展示已知地点
- **AI 实时调试**：面向 NPC/天道叙事规则，支持新增/清空/导入/导出/保存

## 模块化架构

主要加载顺序见 `manifest.json`，当前关键模块：

```text
cyoa-core.js              核心配置、工具、插件设置
cyoa-data.js              数据读写、后端/本地存储回退
cyoa-editor.js            编辑器渲染与保存
cyoa-game-runtime.js      回合、旅行、存档、契约等运行时逻辑
cyoa-game-ai.js           AI 请求适配与相关辅助
cyoa-game-systems.js      材质/生理/附加系统
cyoa-game-constraints.js  约束解析与来源
cyoa-game-prompts.js      叙事 guard prompt 生成
cyoa-game-rules.js        行动/选项约束改写规则
cyoa-game-ui.js           游戏区 UI、输入、选项、互动面板
cyoa-game.js              兼容层（薄封装）
cyoa-main.js              主入口、列表、侧栏、设置
```

> 新功能建议优先新建模块，旧大文件仅做最小挂接。

## 存储与容错

- 保存优先走后端 API
- 失败会回退到 `localStorage`
- 失败弹窗会显示具体错误原因，便于排查数据问题

## 常见问题

### Q1: 保存失败怎么办？

先看弹窗中的具体错误原因。  
若后端异常，插件会自动回退本地保存；刷新后确认数据是否仍在。

### Q2: AI 没按章节/地点来？

确认：

- 当前章节、地点、场景是否配置完整
- 在场角色是否正确
- 世界观/背景文本是否互相冲突

### Q3: 选项显示不稳定？

当前已使用固定列宽与字符截断。若仍拥挤，可调窄按钮字符宽度参数。

## 开发建议

- **提交前**：至少验证“进入游戏 / 对话 / 保存 / 读取 / 地图跳转 / 选项渲染”
- **加新功能**：优先新建 `cyoa-game-*.js` 模块，再由入口挂接
- **改 AI 逻辑**：尽量把“约束规则”和“提示词规则”分离，便于调试
- **避免回归**：任何 UI 调整后都要走一次移动/受限状态场景
