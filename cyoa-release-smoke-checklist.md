# CYOA 发版前回归自检清单（约束体系版）

> 目标：每次打包前，用最短路径确认“面板不丢、输入不挂、约束有效、可回退”。
> 建议：按顺序执行，全部通过再发布。

## 0) 基础环境

- [ ] 强刷页面（`Ctrl+F5`）后进入 CYOA。
- [ ] 打开浏览器控制台，确认无新增红色报错（可有 warning）。

## 1) 游戏面板与输入基本可用

- [ ] 进入游戏后，日志面板正常显示，输入框与发送按钮可用。
- [ ] 发送一条普通输入（例如“我在哪里？”），不出现长期 `...` 卡住。
- [ ] AI 返回后能正常渲染 4 个选项（行动/对话）。

## 2) 温度配置（CYOA 自主管理）

- [ ] 设置页中可见 `AI 对话温度（0-1）` 输入框。
- [ ] 将温度设为 `0.15` 并保存，刷新后值仍保留。
- [ ] 将温度设为 `0.35` 并保存，刷新后值仍保留（说明玩家可自由填写，不强制建议值）。

## 3) 约束档位（strict/balanced/free）

- [ ] 设置页中可见 `约束档位` 下拉。
- [ ] 切到 `strict`，发送一条高风险动作，观察回复更保守。
- [ ] 切到 `balanced`，同类输入下回复相对放松但不越界。
- [ ] 切到 `free`，同类输入下创作自由度提高，仍保留核心协议（4选项/状态变更规范）。

## 4) 自定义 Guard 模板覆盖

- [ ] 在游戏数据中设置 `narrator.guardPromptTemplate` 为自定义文本后生效。
- [ ] 使用 `{{DEFAULT_GUARD}}` 占位时，默认模板会被正确插入。
- [ ] 移除自定义模板后，自动回到档位模板（不报错）。

## 5) Hook 扩展点

- [ ] 配置 `onPlayerInput` 后可改写输入（例如把 action 统一加前缀），流程不崩。
- [ ] 配置 `onAIOutput` 后可改写输出（例如替换词），历史与界面展示一致。
- [ ] Hook 内故意抛异常时，不会中断主流程，仅回退原值继续运行。

## 6) 边界事件协议（CALL_TIAN_DAO）

- [ ] 模拟模型返回 `[CALL_TIAN_DAO] 请求天道裁决：xxx`。
- [ ] 前端应自动转为“天道裁决中”的过渡叙事 + 4个选项，而不是空白。
- [ ] 同时出现系统提示“已接收边界协议”。

## 7) responseMode（默认 text，json 可选）

- [ ] 编辑器“叙述者设定”可见 `回复模式`（text/json）。
- [ ] `text` 模式下行为与旧版一致。
- [ ] `json` 模式下，模型返回合法 JSON 时可正常显示正文/选项并应用 `cyoa_changes`。
- [ ] `json` 模式下，模型返回非法 JSON 时自动回退 text，并显示系统提示（本回合）。

## 8) 开场自动叙述与回归风险点

- [ ] 开始游戏后自动触发开场叙述（无需手动先输入）。
- [ ] 开场回合不应误触发“玩家越权”硬兜底。
- [ ] 切换模型后再次发送，重试链路可工作且不丢面板。

## 9) 存档与历史一致性

- [ ] 一轮对话后刷新页面，历史记录仍可回放。
- [ ] 选项、状态变化、当前节点与显示内容保持一致。
- [ ] 无重复写入或异常跳章/跳地点。

## 10) 发版通过标准

- [ ] 以上 0-9 全部通过。
- [ ] 无阻断级报错（面板消失、输入失效、持续卡 `...`）。
- [ ] 可选功能（json/hook/自定义模板）关闭时，默认体验稳定。

---

## 快速故障定位（30 秒）

1. 先切回 `responseMode=text` 重试。  
2. 再切 `AI_GUARD_PROFILE=strict` 重试。  
3. 暂时清空自定义 `guardPromptTemplate` 与 hooks。  
4. 仍异常则看控制台第一条红错，优先定位 `cyoa-game-ui.js`。
