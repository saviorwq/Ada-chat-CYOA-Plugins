# 密室·富豪的游戏（10分钟冒烟测试）

## 目标

验证导入后“可玩链路”是否完整可用：

- 可进入游戏
- 地图可移动
- 可互动目标面板有内容
- 四选项稳定生成
- 线索任务可推进
- 摩尔斯门规则可触发

## 0. 导入

1. 打开 CYOA 编辑器。
2. 导入 `mishi_fh_99_combined_game.json`。
3. 保存全部后进入游戏。

## 1. 开局检查（1分钟）

- 当前地点应为：`白色房间`
- 当前场景应为：`初醒镜室`
- 右侧面板可见：状态、地图、任务
- 交互区可见：镜子、旧门、地面敲击区

期望：无报错、无空白面板。

## 2. 地图与旅行检查（2分钟）

从 `白色房间 -> 狭长走廊 -> 街道A`：

- 能正常 travel
- 有回合消耗
- 受限步态时 ETA 增加（limitedStepExtraTurns 生效）

期望：地图按钮与方向移动都能推进地点。

## 3. 路人职业反馈检查（2分钟）

在 `街道A / 商业区B / 传媒区C` 进行互动：

- 安保反馈偏秩序
- 媒体反馈偏追问
- 商户反馈偏交易或回避

期望：反馈风格存在差异，不同质化。

## 4. 线索任务推进检查（2分钟）

确认下列任务进展可增长：

- `收集三段诗句线索`
- `验证现实伪装`
- `路人职业画像`

期望：交互后任务状态有变化，且不依赖固定章节硬推进。

## 5. 摩尔斯规则检查（2分钟）

回到 `白色房间`，在门前尝试节律输入：

- 故意输错一次，观察误码反馈（短间隔或长间隔提示）
- 再按正确关键词 `STUART` 对应节律输入

期望：

- 错误输入出现惩罚提示（而非直接答案）
- 正确输入可推进脱离相关任务

## 6. 叙事边界检查（1分钟）

抽查 AI 是否遵守：

- 第一人称视角
- 不引入设定外地点/人物
- 只使用当前场景在场人物

期望：无明显越界叙事。
