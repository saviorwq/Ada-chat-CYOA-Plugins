# 发版检查清单

每次发版前请至少完成以下检查，避免出现“能安装但不好用”的回归。

---

## 1) 代码一致性

- [ ] `AI_DEFINITION_HEARTBEAT_TURNS` 默认值统一（当前目标为 `6`）
- [ ] 无重复函数声明（尤其设置页渲染与入口挂接）
- [ ] 无旧版“AI 自动纠偏重试”执行分支残留
- [ ] 兼容层 `cyoa-game.js` 不引入重复主逻辑

---

## 2) 约束能力

- [ ] 系统模板默认可用，重置按钮可恢复默认模板
- [ ] 约束预设（新手/进阶/硬核）可一键应用且配置持久化
- [ ] 公理注入开关与条数上限生效
- [ ] `guardPromptTemplate` 占位符注入正常（`{{DEFAULT_GUARD}}`、`{{SYSTEM_PROMPT_TEMPLATE}}`）
- [ ] Hook 安全策略生效（危险关键字被拒绝、上下文只读）

---

## 3) 运行链路

- [ ] `responseMode=text/json` 两种模式均可用
- [ ] JSON 解析失败可自动回退 `text` 且显示提示
- [ ] `provider/max_tokens/temperature/top_p` 在请求体正确生效
- [ ] 本地先判定可阻断违规状态变更（仅保留叙事后果）

---

## 4) UI 与交互

- [ ] 游戏面板正常显示（无“消失”或退回旧输入框）
- [ ] 开场自动叙述触发正常
- [ ] 四选项稳定显示（2 行动 + 2 对话）
- [ ] 地图、任务、状态、物品侧栏均可打开且数据正常

---

## 5) 存储与回读

- [ ] 创建新游戏可保存并回读
- [ ] 编辑器各分区（场景/地点/角色/叙述者）保存后可回读
- [ ] 后端失败时可按配置走本地回退（若启用）

---

## 6) 文档一致性

- [ ] `README` 目录与链接可用
- [ ] Wiki 首页入口完整（约束/架构/发布/检查）
- [ ] 作者指南与示例文档引用未失效

---

## 7) 打包与发布

- [ ] 生成 `cyoa-plugin.zip`
- [ ] 生成 `cyoa-plugin.tar.gz`
- [ ] Release 说明包含版本亮点、回归范围、文档入口
